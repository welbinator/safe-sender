# Sprint C1 — Backend Refactor + Alembic + CI Gates + Fail-Fast Defaults

> **For Hermes:** Use `subagent-driven-development` skill to implement this plan task-by-task.

**Goal:** Land the riskiest part of Sprint C first — extract a service/repository layer
from the FastAPI routers, replace ad-hoc raw-SQL migrations with Alembic, kill the CI
test bypass, and fail-fast on insecure DB-URL defaults.

**Architecture:**
- Introduce `backend/services/` (pure business logic, takes a connection) and
  `backend/repositories/` (SQL only, no logic). Routers become thin HTTP adapters.
- Adopt **Alembic** for schema management; baseline against current live schema
  (migration `008_sprint_b_privacy.sql` is the latest applied). Existing raw-SQL
  migrations stay in `backend/migrations/` for history; new changes go through Alembic.
- Remove `|| true` from `.github/workflows/ci.yml` so tests gate merges.
- Make `DATABASE_URL` mandatory in `backend/main.py` (no `changeme` fallback).

**Tech Stack:** FastAPI 0.136, asyncpg 0.29, Alembic 1.13+, pytest, GitHub Actions.

**Out of scope (Sprint C2+):** rate limiting, audit-log table, rule cache, dashboard
TS/Vitest migration, nginx HTTP/2, session-scoped SMTP auth.

## Task 2: Fail-fast on insecure DB URL (H-3)

**Objective:** Remove the `postgresql://sendersafety:changeme@...` fallback in
`backend/main.py`. App must refuse to start if `DATABASE_URL` is unset or contains
a weak password.

**Files:**
- Modify: `backend/main.py:35-40`
- Test: `backend/tests/test_startup_guards.py` (new)

**Step 1: Write failing test**

```python
# backend/tests/test_startup_guards.py
"""Startup-time configuration guards (Sprint C1)."""
import importlib
import os
import sys

import pytest


def _reload_main(monkeypatch, env):
    """Clear cached module and re-import main with patched env."""
    for k in ("DATABASE_URL",):
        monkeypatch.delenv(k, raising=False)
    for k, v in env.items():
        monkeypatch.setenv(k, v)
    sys.modules.pop("main", None)
    return importlib.import_module("main")


def test_missing_database_url_raises(monkeypatch):
    with pytest.raises(RuntimeError, match="DATABASE_URL"):
        _reload_main(monkeypatch, {})


def test_weak_database_url_raises(monkeypatch):
    with pytest.raises(RuntimeError, match="weak"):
        _reload_main(
            monkeypatch,
            {"DATABASE_URL": "postgresql://u:changeme@h/d"},
        )


def test_valid_database_url_accepted(monkeypatch):
    mod = _reload_main(
        monkeypatch,
        {"DATABASE_URL": "postgresql://u:S3cur3-x9!@h/d"},
    )
    assert mod.DATABASE_URL.endswith("/d")
```

**Step 2:** `cd backend && pytest tests/test_startup_guards.py -v` → expect FAIL.

**Step 3: Implement guard**

```python
# backend/main.py (replace lines ~35–40)
_WEAK_DB_PASSWORDS = {"changeme", "secret", "password", "default", "test", ""}
DATABASE_URL = os.environ.get("DATABASE_URL")
if not DATABASE_URL:
    raise RuntimeError(
        "DATABASE_URL is required. Refusing to start with an insecure default."
    )
_pw = urlparse(DATABASE_URL).password or ""
if _pw.lower() in _WEAK_DB_PASSWORDS:
    raise RuntimeError(
        "DATABASE_URL contains a weak/default password. Set a strong password."
    )
```

(Move `from urllib.parse import urlparse` to the top of the file if not already.)

**Step 4:** `pytest tests/test_startup_guards.py -v` → expect 3 passed.

**Step 5:** Commit.
```
git add backend/main.py backend/tests/test_startup_guards.py
git commit -m "feat(backend): fail-fast on missing/weak DATABASE_URL (H-3)"
```

**Pattern (example — customers):**

```python
# backend/repositories/customers.py
"""SQL-only access for the customers table. NO business logic here."""
from databases import Database


async def get_by_id(db: Database, customer_id: str) -> dict | None:
    row = await db.fetch_one(
        "SELECT id, email, plan, domain_verified, created_at "
        "FROM customers WHERE id = :id",
        {"id": customer_id},
    )
    return dict(row) if row else None


async def get_by_email(db: Database, email: str) -> dict | None:
    row = await db.fetch_one(
        "SELECT id, email, plan, domain_verified, created_at "
        "FROM customers WHERE lower(email) = lower(:email)",
        {"email": email},
    )
    return dict(row) if row else None


async def create(db: Database, *, email: str, plan: str) -> dict:
    row = await db.fetch_one(
        "INSERT INTO customers (email, plan) VALUES (:email, :plan) "
        "RETURNING id, email, plan, domain_verified, created_at",
        {"email": email, "plan": plan},
    )
    return dict(row)


# ... mark_domain_verified, set_plan, etc. — extract every SELECT/INSERT/UPDATE
# currently inside routers/customers.py
```

**Tests** (`test_repositories_unit.py`) — use the existing pytest conftest's
TestClient lifecycle which already wires the asyncpg pool. One test per repo
function: insert seed row, call repo function, assert result shape.

## Task 6: Create `services/` layer for customers + rules

**Objective:** Pull business logic out of `routers/customers.py` and `routers/rules.py`

request → call service → return response.

**Files:**

- Create: `backend/services/rules.py`
- Modify: `backend/routers/customers.py`
- Modify: `backend/routers/rules.py`
- Modify: `backend/tests/test_sprint_b_*.py` (only if imports break)
- Create: `backend/tests/test_services_customers.py`
- Create: `backend/tests/test_services_rules.py`

**Pattern (example — service):**

```python
# backend/services/customers.py
"""Customer business logic. HTTP-agnostic — raises ValueError / KeyError, never HTTPException."""
from databases import Database
from repositories import customers as customers_repo


class DomainAlreadyVerified(Exception):
    pass


async def verify_domain(db: Database, customer_id: str, token: str) -> dict:
    customer = await customers_repo.get_by_id(db, customer_id)
    if customer is None:
        raise KeyError("customer not found")
    if customer["domain_verified"]:
        raise DomainAlreadyVerified()
    # ... DNS check, etc. (move from router)
    return await customers_repo.mark_domain_verified(db, customer_id)
```

**Router becomes:**

```python
# backend/routers/customers.py — verify-domain endpoint, after refactor
from services import customers as customers_svc

@router.post("/verify-domain")
async def verify_domain(payload: VerifyDomainPayload, claims=Depends(require_customer)):
    try:
        result = await customers_svc.verify_domain(db, claims["sub"], payload.token)
    except KeyError:
        raise HTTPException(404, "customer not found")
    except customers_svc.DomainAlreadyVerified:
        raise HTTPException(409, "domain already verified")
    return result
```

**Constraint:** Routers may import from `services/` and `repositories/`. Services
may import from `repositories/`. Repositories import nothing project-local.
Document this in `backend/README-migrations.md` (architecture section).

**Tests:** Mock the repo layer with simple fakes — services should be testable
without a DB.

**Verify:** `pytest backend/tests/ -q` → all green, no test regressions.

**Commit:** `refactor(backend): extract services layer for customers + rules`
